How the CAS Logout Works
  1. User-Initiated Logout: When you click "Logout from CAS", you'll be redirected to CAS logout page
  2. CAS Central Logout: CAS will log you out from all connected applications
  3. Back-Channel Logout: CAS sends logout notifications to all registered services
  4. Service Redirect: After logout, CAS redirects you back to this application's home page
  5. Session Cleanup: Your local session is invalidated and you'll need to re-authenticate

Authentication Flow Summary
  1. You accessed the home page without authentication
  2. You clicked "Login with CAS" which redirected you to CAS server
  3. After successful authentication, CAS redirected back with a JWT ticket parameter
  4. The application decoded the JWT token and stored it in your session

  6. When you logout, CAS will handle centralized logout across all services
